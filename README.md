# 📊 Hotel Booking Data Analysis

## 📌 Project Overview

This project analyzes hotel booking data to uncover patterns in **cancellations, seasonal demand, and customer behavior**. The goal is to generate insights that support better decision-making in **revenue management, pricing strategies, and customer targeting**.

---

## 🎯 Objectives

The analysis was guided by the following key questions:

1. Compare cancellation rates between City Hotels and Resort Hotels
2. Identify peak booking months and their cancellation trends
3. Determine whether special requests reduce cancellation likelihood
4. Analyze how guest composition (families vs individuals) affects length of stay
5. Examine the relationship between lead time and previous cancellations

---

## 🗂️ Dataset Description

The dataset contains hotel booking records with variables such as:

* Hotel type (City / Resort)
* Booking dates (arrival month, lead time)
* Cancellation status
* Number of guests (adults, children, babies)
* Special requests
* Previous cancellations

---

## 🛠️ Tools & Technologies

* Microsoft Excel → Data cleaning, pivot tables, correlation analysis
* Power BI → Data visualization and interactive dashboard creation

---

## 🔧 Methodology

The project followed a structured data analysis process:

1. **Data Cleaning & Preparation**

   * Removed inconsistencies and ensured correct data types
   * Created calculated fields (Total Guests, Length of Stay, Is_Family)

2. **Exploratory Data Analysis**

   * Used pivot tables to summarize key metrics
   * Compared trends across hotel types and customer groups

3. **Data Visualization**

   * Built charts (bar, line, combo, scatter) to identify patterns
   * Designed an interactive Power BI dashboard with slicers

4. **Statistical Analysis**

   * Applied correlation analysis to examine relationships between variables

---

## 📊 Key Findings

* **City Hotels** have higher cancellation rates than Resort Hotels
* Booking demand peaks during **summer months**, with higher cancellations
* Guests with **special requests** are less likely to cancel
* **Family groups** tend to stay longer than individual travelers
* A weak positive relationship exists between **lead time and previous cancellations**

---

## 📈 Dashboard Features

* Interactive slicers (hotel type, month, customer segment)
* Visual comparison of bookings and cancellations
* Customer behavior insights (families, special requests)

---

## 💡 Business Implications

* Implement **differentiated cancellation policies** for City Hotels
* Use **dynamic pricing and staffing strategies** during peak seasons
* Encourage **customer engagement (special requests)** to reduce cancellations
* Target **families with long-stay packages**

---

## 🚀 Conclusion

This project demonstrates how data analysis can be used to understand customer behavior and optimize hotel operations. The insights generated can help improve **revenue stability, operational efficiency, and strategic decision-making**.
